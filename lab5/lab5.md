# Лабораторна робота №5. Внутрішня перелінковка

---

## Мета

Навчитись аудитувати внутрішню перелінковку сайту, виявляти типові помилки (orphan pages, надлишкові посилання,
неправильні анкори), будувати схему перелінковки відповідно до silo-структури та впроваджувати виправлення безпосередньо
у проект.

---

## 1. Google Sheets:
### 1.1 "Pages Inventory"  - інвентаризація сторінок

За допомогою ручного аналізу та перевірки вихідного коду (DevTools) було перевірено наявність «сторінок-сиріт». Усі сторінки проєкту мають хоча б одне вхідне посилання з навігації або контенту. Сторінка /about визначена як nav-only, оскільки на неї веде посилання лише з футера.

**[Посилання на Google Sheets (Аркуш "Pages Inverntory")](https://docs.google.com/spreadsheets/d/11QNVKtkD4fDlYjSnt50f4zp1lORtyjCqRMmRoWBDTkk/edit?usp=sharing)**

### 1.2 "Anchor Audit"     - аудит анкорів

**[Посилання на Google Sheets (Аркуш "Anchor Audit")](https://docs.google.com/spreadsheets/d/11QNVKtkD4fDlYjSnt50f4zp1lORtyjCqRMmRoWBDTkk/edit?gid=1348953220#gid=1348953220)**

## 2.  Таблиця глибини кліків для всіх сторінок

Розроблено та зафіксовано повну схему внутрішньої перелінковки, що складається з 20 посилань. Схема базується на принципах Silo-структури: вертикальні посилання (Breadcrumbs), горизонтальні посилання всередині категорій та контекстні посилання з інформаційних статей на комерційні товари.

**[Посилання на Google Sheets (Аркуш "Link Scheme")](https://docs.google.com/spreadsheets/d/11QNVKtkD4fDlYjSnt50f4zp1lORtyjCqRMmRoWBDTkk/edit?gid=865610747#gid=865610747)**

## 3. Заповнений чек-ліст типових помилок

| Помилка                                               | Присутня | Де саме | Як виправити |
|-------------------------------------------------------|----------|---------|--------------|
| Orphan pages                                          | Ні |    -     |  Всі сторінки доступні через головне меню або лістинг блогу.       |
| Generic анкори ("тут", "click here")                  |  Ні |    -     |       Замінити неінформативні слова на ключові запити (напр. "sticker guide").       |
| Посилання на себе (сторінка посилається сама на себе) |  Ні |     -    |       -       |
| Зламані внутрішні посилання (404)                     |  Ні |     -    |        Всі URL перевірено, внутрішні посилання ведуть на робочі сторінки.      |
| Надлишкова перелінковка (10+ посилань на абзац)       |  Ні |      -   |       Кількість посилань у тексті статей відповідає нормі (не більше 3-5 на статтю).       |
| Глибина кліків > 3                                    |  Ні |     -    |        Будь-яка стаття доступна максимум у 2 кліки від головної сторінки.      |
| Посилання через JS (onclick) замість `<a href>`       |  Ні |     -    |      Використовуються стандартні компоненти Next.js Link, які рендеряться як <a href>        |
| Nofollow на внутрішніх посиланнях                     |  Ні |    -     |      Всі внутрішні посилання відкриті для передачі ваги        |
